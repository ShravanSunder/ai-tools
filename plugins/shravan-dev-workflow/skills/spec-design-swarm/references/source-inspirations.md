# Source Inspirations

This skill is original Shravan workflow guidance. These are references for mechanics, not copied text.

- Obra `brainstorming`: context scan, clarifying questions, 2-3 approaches, no implementation before design approval.
- Obra `writing-plans`: file responsibility map, exact paths, validation commands, no placeholders at the planning boundary.
- Claude `feature-dev`, `code-explorer`, `code-architect`: explorer lanes before architecture and competing architecture stances.
- Addy `idea-refine`: divergent to convergent flow, assumptions, MVP shape, non-goals.
- Addy `spec-driven-development`: objective, scope, project structure, commands, test strategy, boundaries, success criteria.
- Dimillian `review-swarm`: shared packet, parallel read-only lanes, parent-owned reducer.
- Matt Pocock `grill-me` and Addy `interview-me`: decision-tree questions with a recommended answer when intent is unclear.

Shravan adaptations:

- No automatic file writes or commits during design.
- Main agent owns synthesis and recommendation.
- Subagents are bounded lanes, not owners.
- Security context is captured as part of the design when sensitive surfaces are touched.
