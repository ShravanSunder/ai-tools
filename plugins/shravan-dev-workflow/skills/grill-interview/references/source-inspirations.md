# Source Inspirations

These are best-practice references, not text sources to copy.

- Matt Pocock `grill-me`: one question at a time, recommended answer with each question, inspect code instead of asking when discoverable.
- Matt Pocock `grill-with-docs`: challenge user language against existing docs and code; surface contradictions immediately.
- Addy Osmani `interview-me`: hypothesis plus confidence, intent extraction, explicit non-goals, and refusal to accept vague approval.
- Addy Osmani `idea-refine`: divergent-to-convergent thinking, hidden assumptions, MVP scope, and Not Doing list.
- Obra `brainstorming` and `writing-plans`: understand before planning, ask only material questions, and keep implementation separate until the artifact is ready.

Shravan adaptation:

- Manual trigger only.
- Integrates with `spec-design-swarm`, `spec-review-council`, `plan-review`, `plan-execute`, and `docs-maintain`.
- Uses `tui-presentation` only when visual structure materially helps.
- Produces decisions and next workflow, not speculative plans.
