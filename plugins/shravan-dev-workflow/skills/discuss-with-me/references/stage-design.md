# Stage: Design

Load this when the discussion is about shaping an idea before a spec or plan exists.

## Focus

- who the work is for
- why now
- what success looks like
- viable approaches and tradeoffs
- non-goals
- hidden assumptions

## Questions To Prefer

```text
Current model:
You want <outcome>, and <artifact/request> is the first shape that came to mind.

My recommended default:
Start with <smallest useful design direction> because <evidence/tradeoff>.

Question:
Is the real goal <outcome>, or is there a different user/job we should optimize for?
```

## Stop Condition

Stop when the user has confirmed the outcome, primary audience, non-goals, and the next artifact or workflow.
