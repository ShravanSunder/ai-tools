# Source Inspirations

These are best-practice references, not text sources to copy.

- Obra `brainstorming`: conversation before construction, material questions only, explicit transition to planning.
- Matt Pocock `grill-me`: one question at a time, recommended answer with each question, inspect code instead of asking when discoverable.
- Matt Pocock `grill-with-docs`: challenge user language against existing docs and code; surface source-of-truth contradictions immediately.
- Addy Osmani `interview-me`: hypothesis plus confidence, intent extraction, explicit non-goals, and refusal to accept vague approval.
- Addy Osmani `idea-refine`: divergent-to-convergent thinking, hidden assumptions, MVP scope, and Not Doing list.
- OpenAI `skill-creator`: lean skill body, one-level references, and trigger-only descriptions.
- Obra `writing-skills`: pressure-test trigger behavior with should-trigger and should-not-trigger scenarios.

Shravan adaptation:

- Manual trigger only.
- Focuses on design, spec, plan, implementation-direction, and docs decisions.
- Keeps debugging, security, review, publishing, and broad research in their own workflow skills.

- Produces shared understanding and workflow routing, not speculative artifacts.
