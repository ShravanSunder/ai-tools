# Source Inspirations

This plugin contains original Shravan workflow skills. These upstream skills are references for best practices, trigger design, failure shields, and output shapes; do not copy their text wholesale.

| Local skill | Source inspirations | Borrowed mechanics | Shravan adaptation |
| --- | --- | --- | --- |
| `implementation-review-swarm` | OpenAI `codex-plugin-cc`, Anthropic PR review toolkit, Dimillian `review-swarm`, Steipete `codex-review`, Sentry `code-review`, Obra `receiving-code-review` | adversarial review, read-only reviewer lanes, reducer verification, severity-first findings, verify-before-fix review reception | Codex subagents are the default/majority backend; `agy` is the substantial-review external model lane; Claude is explicit opt-in through the Claude Code harness; accepted current-session findings are fixed after validation; Oracle is excluded |
| `plan-review` | Obra `writing-plans`, Obra subagent review prompts, Codex adversarial review, Obra `receiving-code-review` | whole-artifact loading, adversarial holes, spec/code comparison, validate-before-addressing feedback | plan review uses bounded reviewer lanes by default for substantial plans, supports opt-in external model lanes, then revises accepted plan issues without implementing code |
| `spec-design-swarm` | Obra `brainstorming`, Claude `feature-dev`, Addy `idea-refine` and `spec-driven-development`, Dimillian `review-swarm`, Matt Pocock `grill-me` | codebase explorer lanes, competing architecture stances, one-question decision discussion, explicit non-goals, parent-owned synthesis | creates design direction before a plan exists; no implementation or automatic commits; security context is captured when sensitive surfaces are touched |
| `discuss-with-me` | Matt Pocock `grill-me` and `grill-with-docs`, Addy `interview-me`, `idea-refine`, `spec-driven-development`, and `doubt-driven-development`, Obra `brainstorming`, OpenAI `skill-creator`, Obra `writing-skills` | thinking clarifier, one-question-at-a-time alignment, recommended answer per question, hypothesis/confidence, steelman, stress-test, non-goals, boundary pressure, inspect code/docs before asking, lean progressive disclosure, trigger eval scenarios | manual trigger only; teaches back and pressure-tests the user's thinking for live design/spec/plan/implementation/docs decisions; routes artifact review, debug, security, publishing, broad research, and docs editing to dedicated workflow skills |
| `orchestrate-goal` | OpenAI Codex `/goal`, Claude `/goal`, curated `define-goal`, Jesse Vincent `rules and gates`, Anthropic agent workflow guidance | durable goal contract, explicit proof gates, visible completion evidence, rules vs gates, thin orchestration over specialist workflows | clear goals become Codex/Claude goal contracts; unclear goals route to `discuss-with-me`; phase work stays in existing skills |
| `docs-maintain` | Cursor Agent Skills `documentation-and-adrs`, Obra `writing-plans`, Obra `verification-before-completion`, Shravan agent-vm and AgentStudio docs patterns, Matt Pocock `grill-with-docs` | docs as decisions, README vs AGENTS vs architecture roles, stale-plan classification, full-artifact validation, source-of-truth reconciliation | keeps AGENTS compact, README human-facing, architecture docs current, and meta-workflow history in docs/changelog |
| `spec-review-council` | Addy `doubt-driven-development`, Obra spec/plan reviewer prompts, Dimillian `review-swarm` | claim/artifact/contract separation, adversarial council lanes, accepted/contested/open synthesis, verify-before-addressing findings | reviews drafted specs/designs before planning; external model lanes are opt-in; lane ownership stays strict |
| `security-router` | OpenAI Codex Security plugin (`security-scan`, `security-diff-scan`, `deep-security-scan`, `fix-finding`) | threat model, discovery, validation, attack-path analysis, scan reports, coverage/candidate ledgers, explicit deferred proof gaps | routes to official Codex Security instead of cloning it; normal review skills keep lightweight security lanes |
| `plan-execute` | Obra `executing-plans`, Obra `subagent-driven-development`, Obra `verification-before-completion` | controller packets, bounded implementer slices, review after slices, verification before done | parent agent owns integration, validation scope, and final completion claims |
| `plan-handoff` | Matt Pocock handoff patterns, Obra planning workflows | copy-pasteable context packet, explicit target audience, temp artifacts | writes repo-local temp handoff plus prints the paste prompt in chat |
| `implementation-handoff` | Matt Pocock handoff patterns, review closeout workflows | implementation-state packet, changed-files context, reviewer prompt | works at planned, in-progress, pre-review, post-review, or blocked stages |
| `debug-investigation` | Obra `systematic-debugging`, Dimillian `bug-hunt-swarm`, Matt Pocock `diagnose`/`triage` | no fixes before root cause, bug packet, read-only investigation lanes, fastest proof path | narrows incident triage into diagnosis-first debugging with main-agent ownership |
| `skill-audit` | Dimillian `project-skill-audit`, Obra `writing-skills`, Sentry `skill-scanner`/`skill-writer` | evidence-backed candidates, update-before-new, pressure-test mindset, concise skill anatomy | audits Shravan session patterns and admired upstream skills before recommending plugin-native changes |
| `tui-presentation` | Shravan TUI output preferences and prior terminal presentation work | visual structure, alignment discipline, diagram-first explanation | bundled here so design/review/debug responses share the same presentation contract |
| `pm-linear-work` | Shravan Linear project organization preferences | docs as truth, tickets as tracking, dependency hygiene | keeps Linear operations plugin-native and separate from review workflows |
