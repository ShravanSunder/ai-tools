# Source Inspirations

This plugin contains original Shravan workflow skills. These upstream skills are references for best practices, trigger design, failure shields, and output shapes; do not copy their text wholesale.

| Local skill | Source inspirations | Borrowed mechanics | Shravan adaptation |
| --- | --- | --- | --- |
| `implementation-review-swarm` | OpenAI `codex-plugin-cc`, Anthropic PR review toolkit, Dimillian `review-swarm`, Steipete `codex-review`, Sentry `code-review`, Obra `receiving-code-review` | adversarial review, read-only reviewer lanes, reducer verification, severity-first findings, verify-before-fix review reception | Codex subagents are the default/majority backend; Claude and Gemini/`agy` are explicit opt-in external lanes; accepted current-session findings are fixed after validation; Oracle is excluded |
| `plan-review-swarm` | Obra `writing-plans`, Obra subagent review prompts, Codex adversarial review, Obra `receiving-code-review` | whole-artifact loading, adversarial holes, spec/code comparison, validate-before-addressing feedback | plan review uses bounded reviewer lanes by default for substantial plans, supports opt-in external model lanes, then revises accepted plan issues without implementing code |
| `spec-design-swarm` | Obra `brainstorming`, Claude `feature-dev`, Addy `idea-refine` and `spec-driven-development`, Dimillian `review-swarm`, Matt Pocock `grill-me` | codebase explorer lanes, competing architecture stances, one-question decision discussion, explicit non-goals, parent-owned synthesis | creates design direction before a plan exists; no implementation or automatic commits; security context is captured when sensitive surfaces are touched |
| `discuss-with-me` | Matt Pocock `grill-me` and `grill-with-docs`, Addy `interview-me`, `idea-refine`, `spec-driven-development`, and `doubt-driven-development`, Obra `brainstorming`, OpenAI `skill-creator`, Obra `writing-skills` | thinking clarifier, one-question-at-a-time alignment, recommended answer per question, hypothesis/confidence, steelman, stress-test, non-goals, boundary pressure, inspect code/docs before asking, lean progressive disclosure, trigger eval scenarios | manual trigger only; teaches back and pressure-tests the user's thinking for live design/spec/plan/implementation/docs decisions; routes artifact review, debug, security, publishing, broad research, and docs editing to dedicated workflow skills |
| `research-swarm` | DeepWiki-style repository Q&A, Codex subagents, Obra parallel agents, Perplexity/Tavily research, Shravan AME and Agent Studio research sessions | question-first research, local re-anchor, bounded subagent lanes, mixed-source routing, evidence ledgers, source-quality labels | gathers evidence into tmp research artifacts; discussion stays with `discuss-with-me`, design synthesis stays with `spec-design-swarm`, and the parent verifies lane claims before decisions |
| `orchestrator-goal` | OpenAI Codex `/goal`, Claude `/goal`, curated `define-goal`, Jesse Vincent `rules and gates`, Anthropic agent workflow guidance | durable goal contract, explicit proof gates, visible completion evidence, rules vs gates, thin orchestration over specialist workflows | clear goals become Codex/Claude goal contracts; unclear goals route to `discuss-with-me`; phase work stays in existing skills |
| `docs-maintain` | Cursor Agent Skills `documentation-and-adrs`, Obra `writing-plans`, Obra `verification-before-completion`, Shravan agent-vm and AgentStudio docs patterns, Matt Pocock `grill-with-docs` | docs as decisions, README vs AGENTS vs durable docs roles, stale-artifact classification, full-artifact validation, source-of-truth reconciliation | keeps AGENTS compact, README human-facing, durable docs current, classifies existing specs/plans/debug artifacts for cleanup or promotion, and stores meta-workflow history in docs/changelog |
| `spec-review-swarm` | Addy `doubt-driven-development`, Obra spec/plan reviewer prompts, Dimillian `review-swarm` | claim/artifact/contract separation, adversarial review lanes, accepted/contested/open synthesis, verify-before-addressing findings | reviews drafted specs/designs before planning; external model lanes are opt-in; lane ownership stays strict |
| `spec-handoff` | Matt Pocock handoff patterns, Obra planning workflows | copy-pasteable context packet, explicit target audience, open questions, source evidence | packages spec/design context before an implementation plan exists; handoff means portability, not approval or completion |
| `plan-create` | Obra `writing-plans`, OpenAI planning guidance, Shravan phase-boundary preferences | implementation task sequence, write surfaces, validation gates, risks, rollback/recovery notes | turns spec/design context into a written implementation plan without editing code, reviewing the plan, or executing it |
| `plan-improve-repo` | `shadcn-improve`, Obra `writing-plans`, Obra `verification-before-completion`, Obra `receiving-code-review` | advisor-only repo audit, plan-as-product, candidate vetting, executable plan template, stale backlog reconciliation | audits repositories into high-leverage improvement plans, validates plan readiness, and routes execution to phase skills instead of editing source |
| `ops-security-review` | OpenAI Codex Security plugin (`security-scan`, `security-diff-scan`, `deep-security-scan`, `fix-finding`) | threat model, discovery, validation, attack-path analysis, scan reports, coverage/candidate ledgers, explicit deferred proof gaps | routes to official Codex Security instead of cloning it; normal review skills keep lightweight security lanes |
| `implementation-execute-plan` | Obra `executing-plans`, Obra `subagent-driven-development`, Obra `verification-before-completion` | controller packets, bounded implementer slices, review after slices, verification before done | parent agent owns integration, validation scope, and final completion claims |
| `plan-handoff` | Matt Pocock handoff patterns, Obra planning workflows | copy-pasteable context packet, explicit target audience, temp artifacts | packages existing implementation plans for review, execution, or continuation; spec/design context routes to `spec-handoff` or `plan-create` |
| `implementation-handoff` | Matt Pocock handoff patterns, review closeout workflows | implementation-state packet, changed-files context, reviewer prompt | packages real branch/diff/validation/blocker state for continuation, review, or audit |
| `debug-investigation` | Obra `systematic-debugging`, Dimillian `bug-hunt-swarm`, Matt Pocock `diagnose`/`triage` | no fixes before root cause, bug packet, read-only investigation lanes, fastest proof path | narrows incident triage into diagnosis-first debugging with main-agent ownership |
| `skill-audit` | Dimillian `project-skill-audit`, Obra `writing-skills`, Sentry `skill-scanner`/`skill-writer` | evidence-backed candidates, update-before-new, pressure-test mindset, concise skill anatomy | audits Shravan session patterns and admired upstream skills before recommending plugin-native changes |
| `tui-presentation` | Shravan TUI output preferences and prior terminal presentation work | visual structure, alignment discipline, disclosure sequence, visual-family selection, semantic markdown separation | bundled here so design/review/debug responses share the same presentation contract without turning diagrams into a Mermaid catalog |
| `ops-linear-tracking` | Shravan Linear project organization preferences | docs as truth, tickets as tracking, dependency hygiene | keeps Linear operations plugin-native and separate from review workflows |
